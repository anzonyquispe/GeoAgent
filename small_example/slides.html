MCP + QGIS Integration

Control professional GIS via natural language

What is Model Context Protocol (MCP)?

Official Definition

MCP is an open standard developed by Anthropic that enables AI assistants to securely connect to external data sources and tools.

Think of it as a USB port for AI - a universal way to plug in any tool or data source.

Key Components

  • MCP Host: The AI application (Claude Desktop, IDE)
  • MCP Server: Exposes tools/data to the AI
  • MCP Client: Connects host to servers
  • Transport: Communication layer (stdio, HTTP)
Claude Desktop / IDE
↕
MCP Protocol (JSON-RPC)
↕
QGIS
Database
APIs

Why MCP?

Instead of building custom integrations for each AI + tool combination, MCP provides a single standard that works everywhere.

How MCP Works (Technical)

1. Server Exposes Tools

# MCP Server defines available tools @mcp.tool() def buffer_analysis( layer: str, distance: float ) -> dict: """Create buffer around features.""" result = qgis.run_buffer(layer, distance) return {"output": result.path} @mcp.tool() def spatial_join( target: str, join_layer: str, predicate: str = "intersects" ) -> dict: """Join layers spatially.""" return qgis.sjoin(target, join_layer)

2. LLM Discovers & Calls Tools

// Claude receives tool definitions { "tools": [ { "name": "buffer_analysis", "description": "Create buffer...", "parameters": { "layer": "string", "distance": "number" } } ] } // Claude decides to call tool { "tool": "buffer_analysis", "arguments": { "layer": "protected_areas", "distance": 5000 } }
Key Insight: The LLM reads tool descriptions and decides when and how to use them based on user's natural language request.

AlphaEarth Code Example

import ee # Initialize Earth Engine ee.Initialize() # Access AlphaEarth embeddings alphaearth = ee.ImageCollection("projects/google/alphaearth_foundations") # Define region of interest (Lima, Peru) roi = ee.Geometry.Rectangle([-77.1, -12.1, -76.9, -11.9]) # Get 2024 embeddings embeddings = alphaearth.filterDate("2024-01-01", "2024-12-31") \ .filterBounds(roi) \ .first() # Sample at building locations buildings = ee.FeatureCollection("users/project/buildings") sampled = embeddings.sampleRegions(collection=buildings, scale=10) # Export for analysis (clustering, prediction, etc.) ee.batch.Export.table.toDrive(collection=sampled, fileFormat="Parquet")
Result: Each building gets a 64-dimensional embedding capturing its spatial characteristics. Use for clustering building types, predicting ages, or finding similar neighborhoods.

What is an Embedding? (Deep Dive)

Simple Definition

An embedding is a way to represent complex data (images, text, locations) as a list of numbers that a computer can understand and compare.

Analogy: GPS Coordinates

Just like GPS uses 2 numbers (lat, lon) to represent any location on Earth, embeddings use 64 numbers to represent the "characteristics" of a place.

Key Insight

Similar places have similar numbers. Two urban areas will have embeddings that are close together, while a forest and a city will be far apart.

Visual Example

// Urban pixel embedding [0.82, 0.75, 0.91, 0.88, 0.12, 0.08, ...] ↑ High values = urban features // Forest pixel embedding [0.11, 0.09, 0.15, 0.12, 0.89, 0.92, ...] ↑ Low urban ↑ High vegetation // Similarity calculation cosine_similarity(urban, forest) = 0.15 cosine_similarity(urban1, urban2) = 0.94

The neural network learns which dimensions capture which features automatically - we don't program the rules!

How Does the AI Create Embeddings?

Satellite Image (10m x 10m pixel)
↓
Convolutional Neural Network (CNN)
↓
64 Numbers (Embedding Vector)

Training Process

  1. Input: Millions of satellite images
  2. Task: Predict what's in neighboring pixels
  3. Learning: Network discovers patterns
  4. Result: Meaningful 64-dim representation

Self-Supervised Learning

The model learns without labels. It predicts masked parts of images, forcing it to understand:

  • Land cover patterns
  • Building shapes and density
  • Road networks
  • Vegetation types
  • Water bodies

Analogy: Word Embeddings (Word2Vec)

Spatial embeddings work like word embeddings in NLP - a concept your students may already know!

Word Embeddings (NLP)

king → [0.2, 0.8, 0.1, ...]

queen → [0.3, 0.7, 0.1, ...]

apple → [0.9, 0.1, 0.8, ...]

Famous example:
king - man + woman ≈ queen

Words with similar meanings have similar vectors!

Spatial Embeddings (GeoAI)

Lima downtown → [0.9, 0.8, 0.2, ...]

Miraflores → [0.85, 0.75, 0.25, ...]

Amazon forest → [0.1, 0.2, 0.95, ...]

Spatial equivalent:
Lima - Peru + Colombia ≈ Bogotá

Places with similar characteristics have similar vectors!

Key Concept: Embeddings capture semantic meaning in numbers, enabling mathematical operations on concepts.

What Do the 64 Dimensions Represent?

Each dimension captures different spatial characteristics. The model learns these automatically!

DimsLikely CapturesExample
1-10Urban densityBuildings, roads, concrete
11-20Vegetation healthNDVI-like green patterns
21-30Water presenceRivers, lakes, ocean
31-40Texture patternsRegular vs irregular
41-50Elevation hintsShadows, terrain
51-64Temporal signalsSeasonal changes

* These are illustrative - actual dimensions are learned, not predefined

Why 64 Dimensions?

  • Too few (e.g., 8): Can't capture complexity
  • Too many (e.g., 512): Computationally expensive
  • 64: Good balance of detail vs efficiency

Embeddings vs Traditional Classification

Aspect Traditional Classification Foundation Model Embeddings
Output Categories (urban, forest, water) 64-dimensional vector
Training Data Needs labeled examples Self-supervised (no labels)
New Tasks Retrain from scratch Transfer learning (reuse embeddings)
Nuance "Urban" or "not urban" (binary) Captures gradients and subtypes
Comparison Same category = same Continuous similarity score
Scale Project-specific Global, reusable
Key Advantage: Embeddings are general-purpose. Train once, use for many downstream tasks (classification, regression, clustering, search).
